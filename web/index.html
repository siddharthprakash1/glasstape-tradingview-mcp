the upgrade

The community proved the idea. We made it hold up.

The original bridge is brilliant and fragile — untyped JavaScript poking undocumented TradingView internals, breaking silently on every update. Glasstape keeps the magic and engineers the parts that hurt.

quickstart

Running in two commands.

Launch TradingView with the debug port, point Claude at the server, and you're live. All local — no TradingView data ever leaves your machine.

1 — launch TradingView
# macOS · enable the local debug port
open -a "TradingView" --args \
  --remote-debugging-port=9222

# then confirm the bridge can see it
npx glasstape health
2 — connect Claude
// .mcp.json
{
  "glasstape": {
    "command": "npx",
    "args": ["glasstape", "serve"]
  }
}

// now just ask Claude →
"switch to ETHUSD, 1D, add MACD"
